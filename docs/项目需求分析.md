## 使用说明

本文档适用于两种场景：

- **新项目启动**：团队有一个新想法，尚未形成结构化需求，通过 `/requirements-analysis` 引导填写。
- **已有项目新增功能**：项目已运转，需新增功能模块或迭代目标，通过 `/requirements-analysis` 关联现有架构和链路后填写。

填写完成后，后续所有开发工作流（`/task-intake` -> 实现 -> `/pre-pr-check` -> `/merge-check`）均以本文档为起点。

本文档如有变更（如新增功能、调整范围、修改成功标准），必须同步更新本文件。
