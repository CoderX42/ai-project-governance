The Problem

Vibe Coding 常见失控场景

🧠

AI 擅自改代码、不读文档

每次对话 AI 都凭"记忆"行动，不读取项目规范，随意改动核心模块。

📦

PR 混入调试代码

console.log、临时注释、无关格式改动全部带进 Pull Request，review 负担大幅增加。

📄

改结构不更新文档

代码改了，文档没动。一个月后没有人知道项目的真实状态。

🔀

AI 开发流程随意跳步

环境检查还没做就直接写代码，PR 还没过审就合并，7 阶段流程被随意跳过。

🚫

乱码视为正常不阻塞

中文文档、日志、注释出现乱码不修复，直接提交，导致协作时信息失真。

📝

文档与代码不同步

代码改了但文档没动，AI 读到的规范与实际行为不符，项目状态逐步失控。

Core Features

覆盖 AI 协作全生命周期的治理能力

📋

7 个 Skill Action

内置 project-init、task-intake、pre-pr-check、merge-check、requirements-analysis、governance-propose、governance-help，完整覆盖从初始化到合并的全流程。

📄

5 份治理文档

CLAUDE.md、AI开发与PR流程.md、项目开发规范.md、项目完整链路说明.md、项目文件结构说明.md，结构清晰、互不重复。

⚡

一键初始化脚本

远程 curl 一行命令，自动复制文档、替换占位符、配置 Cursor Skill，无需手动操作。

🔒

PR 流程硬约束

7 阶段强制推进：环境检查 → 基线同步 → 开发 → Pre-PR 检查 → 创建 PR → 授权合并 → 验证。禁止跳步。

🌐

✅

Architecture

治理规则分层设计

1

总章程 — CLAUDE.md

永久生效的核心规则，定义 AI 不能做什么、必须遵守什么。AI 每次工作前必须阅读。

2

细则 — 4 份根文档

AI开发与PR流程.md（流程）、项目开发规范.md（约束）、项目完整链路说明.md（架构）、项目文件结构说明.md（索引）。

3

流程化 — Skill / Prompt

可调用、可复用的工作流入口。7 个 action 覆盖完整生命周期，AI 可主动触发。

4

硬拦截 — Hook（可选）

强制执行关键检查点，如在发 PR 前自动拦截分支错误、diff 问题、文档未同步等场景。
