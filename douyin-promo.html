7 阶段 PR 铁律

01

环境检查

gh、git 认证、远程配置、分支状态

02

同步基线

切到 dev → 拉最新 → 创建功能分支

03

开发

先读文档再改代码，不猜测、不跳步

04

PR 前同步

检查基线是否过时，过时就 rebase

05

提交与推送

只含任务相关改动，无调试代码

06

创建 PR

目标 dev，结构化正文

07

授权合并

明确授权才合并，分支自动删除
